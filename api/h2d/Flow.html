Setting a background tile will create a ScaleGrid background which uses the borderWidth/Height values for its borders. It will automatically resize when the reflow is done to cover the whole Flow area.

When set to true, the debug will display red box around the flow, green box for the client space and blue boxes for each element.

Set enableInteractive to true to create an h2d.Interactive accessible through the interactive field which will automatically cover the whole Flow area.

Calculate the client width, which is the innner size of the flow without the borders and padding.

read onlyinteractive:Interactive

See enableInteractive.

isInline:Bool

When isInline is set to false, the flow size will be reported based on its bounds instead of its calculated size. See getSize() documentation.

isVertical:Bool

By default, elements will be flowed horizontaly, then in several lines if maxWidth is reached. You can instead flow them vertically, then to next column is maxHeight is reached by setting the isVertical flag to true.

lineHeight:Null<Int>

minHeight:Null<Int>

multiline:Bool

When set to true, uses specified lineHeight/colWidth instead of maxWidth/maxHeight for alignment.

needReflow:Bool

If some sub element gets resized, you need to set reflow to true in order to force the reflow of elements. You can also directly call reflow() which will immediately update all elements positions.

If a reflow is needed, reflow() will be called before rendering the flow. Each change in one of the flow properties or addition/removal of elements will set needReflow to true.

Flow total width (inlcudes borders and paddings)

overflow:Bool

Enabling overflow will treat maxWidth/maxHeight and lineHeight/colWidth constraints as absolute : bigger elements will overflow instead of expanding the limit.

Methods

addSpacing (v:Int):Void

Adds some spacing by either increasing the padding of the latest non absolute element or the padding of the flow if no element was found. The padding affected depends on the isVertical property.

Returns the bounds of the sprite for its whole content, recursively. If relativeTo is null, it will return the bounds in the absolute coordinates. If not, it will return the bounds relative to the specified sprite coordinates. You can pass an already allocated bounds or getBounds will allocate one for you and return it.

This is similar to getBounds(parent), but instead of the full content, it will return the size based on the alignement of the Sprite. For instance for a text, getBounds will return the full glyphs size whereas getSize() will ignore the pixels under the baseline.
